# HIS报表查询系统 - 部署说明

## 环境要求

- **JDK 17+**（推荐 JDK 21，Eclipse Adoptium）
- **Oracle Database 11g+**
- Windows Server 2016+ / Linux

## 目录结构

```
sql-report-server-clean/
├── sql-report-server-1.0.0.jar   # 主程序
├── 启动服务.bat                    # Windows启动脚本
├── 部署说明.md                     # 本文档
├── .env.example                   # 环境变量示例
├── scripts/                       # 报表SQL文件目录
│   └── *.sql                      # 报表定义文件
├── data/                          # 运行时数据（自动生成）
│   ├── datasources.json           # 数据源配置
│   ├── report-groups.json         # 报表分组配置
│   └── users.json                 # 用户数据
└── logs/                          # 日志目录（自动生成）
    └── his-report-server.log      # 运行日志
```

## 快速部署

### 方式一：Windows 双击启动

1. 编辑 `启动服务.bat`，修改以下配置：
   - `DB_URL` - Oracle连接地址
   - `DB_USERNAME` - 数据库用户名
   - `DB_PASSWORD` - 数据库密码
   - `ADMIN_PASSWORD` - 管理员密码（至少8位）

2. 双击 `启动服务.bat` 启动

### 方式二：命令行启动

```bash
java -jar sql-report-server-1.0.0.jar \
  --spring.datasource.url=jdbc:oracle:thin:@你的数据库地址:1521:SID \
  --spring.datasource.username=数据库用户名 \
  --spring.datasource.password=数据库密码 \
  --app.security.admin.password=管理员密码
```

### 方式三：环境变量

```bash
# 复制并编辑环境变量文件
cp .env.example .env
# 编辑 .env 填入实际值

# Linux
export DB_URL=jdbc:oracle:thin:@host:1521:sid
export DB_USERNAME=scott
export DB_PASSWORD=tiger
export ADMIN_PASSWORD=your_password
java -jar sql-report-server-1.0.0.jar

# Windows (cmd)
set DB_URL=jdbc:oracle:thin:@host:1521:sid
set DB_USERNAME=scott
set DB_PASSWORD=tiger
set ADMIN_PASSWORD=your_password
java -jar sql-report-server-1.0.0.jar
```

## 访问系统

- 地址：http://localhost:8080
- 用户名：`admin`
- 密码：启动时配置的 `ADMIN_PASSWORD`

## 添加报表

在 `scripts/` 目录下创建 `.sql` 文件，格式：

```sql
---
name: 报表名称
description: 报表描述
dataSource: default
group: default
params:
  - name: start_date
    label: 起始日期
    type: date
    required: true
  - name: dept_name
    label: 科室名称
    type: text
    default: "%"
---

SELECT * FROM your_table WHERE date >= :start_date AND dept LIKE :dept_name
```

**参数类型**：`date`、`text`、`number`、`select`

**自动转换**：SQL中的 `[参数名]` 会自动转为 `:param_name` 格式

## 数据源管理

系统支持多数据源配置，管理员可在 Web 界面管理：

1. 登录管理后台 → 数据源管理
2. 添加数据源（填写连接信息）
3. 测试连接 → 设为默认
4. 报表可通过 YAML 头部 `dataSource` 字段绑定特定数据源

数据源配置存储在 `data/datasources.json`。

## 生产环境优化

### JVM 参数

```bash
java -Xms2g -Xmx4g -XX:+UseG1GC -XX:MaxGCPauseMillis=200 \
  -XX:+HeapDumpOnOutOfMemoryError -XX:HeapDumpPath=./logs \
  -jar sql-report-server-1.0.0.jar ...
```

### 连接池调优

```bash
--spring.datasource.hikari.maximum-pool-size=50
--spring.datasource.hikari.max-lifetime=600000
--spring.datasource.hikari.leak-detection-threshold=120000
```

### 查询超时

```bash
--report.query-timeout=300
```

## 常见问题

### ORA-17008 连接超时

数据库连接被服务端关闭。调整 `max-lifetime`（应小于数据库 `SQLNET.EXPIRE_TIME`）：
```bash
--spring.datasource.hikari.max-lifetime=600000
```

### 查询超时

复杂查询超过默认120秒限制：
```bash
--report.query-timeout=300
```

### 连接池耗尽

并发量大时增加连接池：
```bash
--spring.datasource.hikari.maximum-pool-size=50
```

### 端口冲突

修改启动端口：
```bash
--server.port=8081
```

## 日志

运行日志位于 `logs/his-report-server.log`，自动轮转（单文件50MB，保留30天，总上限1GB）。

审计日志包含操作类型、用户、IP、耗时等信息，格式：
```
[AUDIT] action=query, report=xxx, user=admin, ip=127.0.0.1, duration=120ms
```
